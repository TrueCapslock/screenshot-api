## v1.0.16 (2026-05-15)
- Sentry: moved DSN from hardcoded in `src/instrument.js` to `SENTRY_DSN` env var in `.env`; Sentry init is skipped if DSN is unset (`src/instrument.js`, `.env`)
- Dev menu: Sentry release now reads `SENTRY_ORG` and `SENTRY_PROJECT` from `.env` (was using hardcoded defaults `anomaly`/`screenshot-api` which caused 401) (`dev-menu.sh`)
- Dev menu: always load `SENTRY_AUTH_TOKEN` from `.env` instead of only when env var is unset, preventing stale shell env vars from overriding (`dev-menu.sh`)
- Dev menu: trim whitespace from values read from `.env` via `xargs` to avoid trailing-space bugs (`dev-menu.sh`)
- Compare: when comparing fullpage vs non-fullpage screenshots, crop to the shorter height and the wider width instead of stretching both images — uses `Math.min` for height and `sharp` `fit: 'cover'` with `position: 'top'` (`src/routes/compare.js`)
- Compare slider UI: size the slider from the generated diff image dimensions when available, with absolutely positioned baseline/current images cropped from the top so fullpage images no longer determine the slider height (`public/index.html`)
